## Описание

Программный интерфейс на Python для управления коллаборативными роботами серии
RC. Актуальная документация на API доступна по [ссылке](https://robopro.pro/doc).

## Установка проекта

### Для пользователей

Установить проект для использования можно несколькими способами. Способы установки можно разделить на два типа:
1. установка проекта как пакета python;
2. установка зависимостей проекта.

Первая группа способов позволяет использовать пакет как любой другой пакет
python, второй же - использовать исходный код пакета.

Установка проекта как пакета:
  - [Установка пакета из zip архива](#установка-пакета-из-zip-архива)
  - [Установка пакета в виртуальную среду](#установка-пакета-в-виртуальную-среду)

Установка зависимостей проекта:
  - [Автоматически с использованием uv](#установка-зависимостей-с-использованием-uv);
  - [В виртуальную среду](#установка-зависимостей-в-виртуальную-среду);
  - [На системный интерпретатор python](#установка-зависимостей-на-системный-интерпретатор-python).

Выбор конкретного метода стоит производить исходя из решаемых задач, в
большинстве случаев требуется установка проекта как пакета.

#### Установка пакета из zip архива

Установить пакет можно с помощью поставляемого zip архива.

Для установки на системный интерпретатор python необходимо выполнить команду:
```bash
pip install <path/to/archive>.zip
```
где `<path/to/archive>` - путь до архива с пакетом.

В некоторых дистрибутивах Linux установка пакетов на системный интерпретатор
python без указания дополнительных флагов запрещена (для защиты системы). В
таких случаях рекомендуется использовать виртуальное окружение и поставить пакет
в него с помощью описанной выше команды.

#### Установка пакета в виртуальную среду

Необходимо создать виртуальную среду с версией python 3.10 - 3.14 (любым удобным
вам способом (`python -m venv`, `poetry`, `uv`, ...)), активировать ее и
установить в нее пакет.

Если имеется `zip` архив проекта, то установить в нее пакет можно командой (в
менеджерах `poetry`, `uv` и т.п. часто имеется своя команда для этого):
```bash
pip install <path/to/archive>.zip
```

Если `zip` **архива** проекта **нет**, то в команде выше необходимо указать путь
до директории с проектом.
```bash
pip install <path/to/directory>
```

В конце, необходимо получить экземпляр терминала, из которого следует запускать
исполняемые файлы (активировать виртуальную среду). Для активации виртуальной
среды следует выполнить команду:

```bash
source ./.venv/bin/activate # Для bash, zsh и им подобных (Linux/macOS)
source ./.venv/bin/activate.fish # Для fish (Linux/macOS)
.venv\Scripts\activate.bat     # Windows
.venv\Scripts\activate     # Windows (иногда запрещено системой)
```

#### Установка зависимостей с использованием uv

Данный метод позволяет установить и использовать пакет в пределах виртуальной
среды, созданной uv.

Сначала необходимо установить
[uv](https://docs.astral.sh/uv/getting-started/installation/) - менеджер
проектов python - любым удобным для вас способом. Это можно сделать, например,
следующей командой:
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

После этого установить зависимости проекта можно с помощью команды:

```bash
uv sync --no-dev
```

В конце, необходимо получить экземпляр терминала, из которого следует запускать
исполняемые файлы (активировать виртуальную среду). Для активации виртуальной
среды следует выполнить команду:

```bash
source ./.venv/bin/activate # Для bash, zsh и им подобных (Linux/macOS)
source ./.venv/bin/activate.fish # Для fish (Linux/macOS)
.venv\Scripts\activate.bat     # Windows
.venv\Scripts\activate     # Windows (иногда запрещено системой)
```

После этого пакет и его зависимости будут доступны для использования в рамках
созданной виртуальной среды.

#### Установка зависимостей в виртуальную среду

Необходимо создать виртуальную среду с версией python 3.10 - 3.14 (любым удобным
вам способом) и установить в нее необходимые зависимости. Для установки
зависимостей вручную в терминале необходимо выполнить команду:

```bash
pip install -r requirements.txt
```

#### Установка зависимостей на системный интерпретатор python

Установить зависимости на системный интерпретатор python можно с помощью команды:

```bash
pip install -r requirements.txt
```

Важно заметить, что проект совместим с версиями python от 3.10 до 3.14
включительно. Если версия установленного в вашей системе интерпретатора отлична
от указанных, то работоспособность проекта не гарантируется.

В некоторых дистрибутивах Linux установка пакетов на системный интерпретатор
python без указания дополнительных флагов запрещена (для защиты системы).

### Для разработчиков

> Под разработчиками понимаются непосредственно разработчики пакета API. Если
> Вы не собираетесь вносить изменения в пакет API, то следует использовать
> один из методов установки, описанный в разделе
> [Для пользователей](#для-пользователей).

Для комфортного процесса разработки рекомендуется настроить инфраструктуру для 
разработки.

Предварительно необходимо установить менеджер проектов
[uv](https://docs.astral.sh/uv/getting-started/installation/). Сделать это можно
либо по инструкции с официального сайта, либо с помощью команды
```bash
make install
```

Далее необходимо инициализировать виртуальную среду следующей командой:

```bash
uv sync
```

Она создаст виртуальную среду python нужной версии и установит все зависимости
проекта (в том числе и для разработки). Выбрать конкретную версию python можно
путем указания желаемой версии интерпретатора в файле
[.python-version](.python-version).

Далее работать следует исключительно из виртуальной среды, терминал с которой
можно получить с помощью команды:

```bash
source ./.venv/bin/activate # Для bash, zsh и им подобных (Linux/macOS)
source ./.venv/bin/activate.fish # Для fish (Linux/macOS)
.venv\Scripts\activate.bat     # Windows
.venv\Scripts\activate     # Windows (иногда запрещено системой)
```

Наконец, следует установить pre-commit хук на репозиторий.
Это добавит процедуру форматирования измененных исходных файлов, запуск линтера
перед каждым коммитом и обновление [requirements.txt](requirements.txt) при
необходимости.

```bash
pre-commit install
```

## Запуск проекта

Проект не предполагает запуск, так как не содержит готовых к запуску скриптов.
При необходимости запустить написанный самостоятельно скрипт следует
придерживаться следующего порядка действий:
1. Установить [зависимости](#для-пользователей)
2. Запустить ваш скрипт одним из следующих способов:
   - активировать виртуальную среду, а затем запустить скрипт командой
      `python <path/to/script.py>`
   - выполнить команду `uv run <path/to/script.py>`, если вы используете `uv`
   - запустить скрипт командой `python3 <path/to/script.py>` если вы используете
      системный интерпретатор python

## История изменений

Полная история изменений проекта ведётся в файле [CHANGELOG.md](CHANGELOG.md) в
соответствии с форматом [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
проект придерживается
[Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Для быстрого ознакомления с ключевыми изменениями рекомендуется изучить
историю изменений в файле [CHANGELOG.md](CHANGELOG.md), где изменения
сгруппированы по типам:
- *Added* — новые функции и возможности
- *Changed* — обратно-несовместимые или совместимые изменения
- *Fixed* — исправления ошибок
- *Removed* — уделенные компоненты

Текущая версия проекта указана в файле [pyproject.toml](pyproject.toml) в поле
`version`, например:
```toml
[project]
name = "API"
version = "1.4.2" # <--- вот версия проекта
requires-python = ">=3.10,<3.15"
description = "Python programming interface for..."
```

## Тестирование проекта (для разработчиков)

В проекте имеются тесты основных методов API, позволяющие удостовериться в
совместимости ядра контроллера робота с командами API. Для их запуска
необходимо:
1. Установить [зависимости](#для-разработчиков)
2. Запустить локально ядро робота (либо не локально, для этого необходимо
изменить целевой IPv4 адрес робота в файле
[tests/conftest.py](tests/conftest.py), но так делать не рекомендуется)
3. Запустить тесты с помощью одной из команд:
   -  `make test` - запуск тестов на версии python текущей виртуальной среды
   -  `make test-<version>` (например, `make test-3.11`)- запуск тестов на
указанной версии python (виртуальная среда `uv` будет пересоздана)
   -  `make test-all` - запуск тестов на всех поддерживаемых версиях python
(см. `PYTHON_VERSIONS` в [Makefile](Makefile) виртуальная среда `uv` будет
пересоздана)

## Описание сценариев make (для разработчиков)

Основные сценарии:
* `make lint` - линтинг исходных файлов
* `make format` - автоформатирование исходных файлов
* `make docs` - автогенерация интерактивной документации к API
* `make test` - запуск интеграционных тестов публичных методов API для текущей
версии python
* `make test-all` - запуск интеграционных тестов публичных методов API для всех
совместимых версий python

Полный список доступных сценариев и их описание можно получить с помощью команды
`make help` или же просто `make`.
